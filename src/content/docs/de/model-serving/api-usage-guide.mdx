## Einleitung
Das T-Systems LLM Serving verfuegt ueber eine API-Schnittstelle, die mit den Chat-, Completion-, Vision-, Audio- und Embedding-APIs von OpenAI kompatibel ist.
Es gewaehrleistet eine nahtlose Integration und Funktionalitaet, die den Moeglichkeiten der urspruenglichen API entspricht.
Die folgenden Tutorials zeigen Ihnen, wie Sie das Python-Paket mit unserem Service verwenden.

Fuer RAG-Anwendungsfaelle schauen Sie sich bitte auch unsere Abschnitte zur Llama-Index- und LangChain-Integration an.

## HTTPS-API-Referenz
Wenn Sie stattdessen die rohen POST/GET-API-Endpunkte nachschlagen moechten, weil Sie eine andere Programmiersprache verwenden moechten, schauen Sie sich bitte die HTTPS-API-Referenz [hier](https://llm-server.llmhub.t-systems.net/redoc) an.

### Completion-API 

**Bei dieser API wird der Rohtext direkt an das LLM gesendet, ohne dass ein LLM-spezifisches Tag-Template erforderlich ist.**

<Tabs>
  <TabItem label="Python">

```python
model = "Llama-3.3-70B-Instruct"  # choose one of the available LLMs (so no embedding or speech model)

    model=model,
    prompt="What is the Python programming language?",
    stream=stream,
    temperature=0.2,
    max_tokens=128
)

if not stream:
    print(completion.choices[0].text)

else:
    for chunk in completion:
        if chunk.choices:
            if chunk.choices[0].text is not None:
                print(chunk.choices[0].text, end="", flush=True)
```

:::note
Die Completions-API ist nur fuer Open-Source-Modelle verfuegbar. Um das korrekte Verhalten der Modelle zu erhalten, muessen Sie eine bestimmte Reihe von Templates befolgen, die der Autor waehrend der Instruction-Fine-Tuning-Phase des LLM verwendet hat (z.B. `## System`, `## Assistant`).

#### Audio-Uebersetzung  

Die Uebersetzungs-API uebersetzt Audioinhalte ins Englische und transkribiert sie gleichzeitig.

<Tabs>
  <TabItem label="Python">
    ```python
    import time

    audio_file_path = "/path/to/audio_file.mp3"  # Path to your audio file

    start = time.time()
    with open(audio_file_path, "rb") as audio_file:
        translation = client.audio.translations.create(
            model="whisper-1",
            file=audio_file,
            temperature=1.0
        )
    stop = time.time()

    print("\nTime taken: ", stop - start)
    print("Translation:", translation)
    ```

```
Time taken: 8.66 seconds

```

:::note  
- **Anwendungsfall Uebersetzung**: Die Uebersetzungs-API ist nuetzlich, um nicht-englisches Audio fuer Barrierefreiheit und Analyse ins Englische zu konvertieren.  
- **`temperature`**: Hoehere Werte (z.B. `1.0`) foerdern vielfaeltigere Ausgaben, waehrend niedrigere Werte (z.B. `0.0`) deterministische Ergebnisse erzeugen.  
:::

#### Hauptmerkmale der Audio-APIs  

1. **Automatische Spracherkennung**: Vereinfacht die Verarbeitung durch automatische Identifizierung der Eingangssprache.  
2. **Anpassbare Ausgabe**: Passen Sie das Transkriptions- und Uebersetzungsverhalten mit Parametern wie `language` und `temperature` an.  
3. **Effiziente Verarbeitung**: Geringe Latenz sowohl fuer Transkriptions- als auch fuer Uebersetzungsaufgaben.

### Reasoning-Steuerung

:::note
Fuer unterstuetzte Modelle (o1, o1-mini, o3, o3-mini, o4-mini, Gemini 2.5, Claude 3.7, Claude 4) kann das Reasoning ueber den Parameter reasoning_effort gesteuert werden. Der Parameter kann auf die Werte "low", "medium" und "high" gesetzt werden.
:::

<Tabs>
  <TabItem label="Python">
    ```py
    model = "claude-sonnet-4"
    stream = True
    system_prompt = """You are an AI assistant named {model}. You are truthful, concise, and helpful. Always respond in a friendly and professional manner."""
    user_prompt = "What is your name?"
    
    start=time.time()
    chat_response = client.chat.completions.create(
        model=model,
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt},
        ],
        stream=stream,
        reasoning_effort="low" # you can set either "low","medium","high"
    )
    
    count = 0
    if not stream:
        print("Answer: ",chat_response.choices[0].message.content)
    else:
        for chunk in chat_response:
            if chunk.choices:
                if chunk.choices[0].delta.content is not None:
                    print(chunk.choices[0].delta.content, end="", flush=True)
                    count+=1
    ```
  </TabItem>
</Tabs>

:::note
Fuer Qwen3 kann das Reasoning durch Hinzufuegen des Schluesselworts "/no_think" zum Prompt deaktiviert werden.
:::

<Tabs>
  <TabItem label="Python">
    ```py
    model = "Qwen3-30B-A3B-FP8"
    stream = True
    system_prompt = """You are an AI assistant named {model}. You are truthful, concise, and helpful. Always respond in a friendly and professional manner."""
    user_prompt = "What is your name?"
    
    start=time.time()
    chat_response = client.chat.completions.create(
        model=model,
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": "/no_think " + user_prompt},
        ],
        stream=stream
    )
    
    count = 0
    if not stream:
        print("Answer: ",chat_response.choices[0].message.content)
    else:
        for chunk in chat_response:
            if chunk.choices:
                if chunk.choices[0].delta.content is not None:
                    print(chunk.choices[0].delta.content, end="", flush=True)
                    count+=1
    ```
